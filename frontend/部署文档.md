# 前端部署文档（ECS + 本地后端）

## 架构

```
用户浏览器 → ECS服务器(nginx) → 静态文件
                ↓ API请求
          本地电脑(:8080) → Spring Boot 后端
```

- 前端：ECS 服务器，nginx 托管打包后的静态文件
- 后端：本地电脑，Spring Boot 运行在 8080 端口

---

## 一、配置 API 地址

编辑 `frontend/.env.production`，将 `你的公网IP` 替换为本地电脑的公网 IP：

```properties
VITE_API_BASE_URL=http://公网IP:8080/api
VITE_WS_HOST=公网IP:8080
```

> 如何查公网 IP：浏览器搜索"我的IP"，或在本地终端执行 `curl ifconfig.me`

---

## 二、打包前端

```bash
cd E:\java\ai-interview\frontend
npm run build
```

打包产物在 `dist/` 目录，将整个 `dist/` 上传到 ECS。

---

## 三、ECS 配置 nginx

### 3.1 安装 nginx

```bash
# Ubuntu/Debian
sudo apt update && sudo apt install nginx -y

# CentOS
sudo yum install nginx -y
```

### 3.2 上传文件

将 `dist/` 目录上传到 ECS，例如 `/home/deploy/dist/`。

可以用 scp：
```bash
scp -r dist/ root@ECS公网IP:/home/deploy/dist/
```

### 3.3 配置 nginx

```bash
sudo vim /etc/nginx/conf.d/interview.conf
```

写入以下内容：

```nginx
server {
    listen 80;
    server_name _;

    # 前端静态文件
    root /home/deploy/dist;
    index index.html;

    # SPA 路由回退（Vue Router history 模式必须）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存（带 hash 的文件长期缓存）
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 3.4 启动 nginx

```bash
# 测试配置
sudo nginx -t

# 启动/重启
sudo systemctl restart nginx
sudo systemctl enable nginx
```

### 3.5 ECS 安全组

在云服务器控制台的安全组中，放行 **80 端口**（HTTP）。

## 四、本地电脑配置

### 4.1 Windows 防火墙放行 8080

1. 打开"Windows Defender 防火墙" → "高级设置"
2. 左侧点"入站规则" → 右侧点"新建规则"
3. 选"端口" → TCP → 特定端口填 `8080`
4. 选"允许连接" → 完成

### 4.2 路由器端口转发（如果在 NAT 后面）

登录路由器管理页面（通常 `192.168.1.1`），找到"端口转发"或"虚拟服务器"：

| 外部端口 | 内部IP | 内部端口 | 协议 |
|---------|--------|---------|------|
| 8080 | 本机局域网IP | 8080 | TCP |

### 4.3 启动后端

```bash
cd E:\java\ai-interview\backend
# 启动 Spring Boot
mvn spring-boot:run
```

确认后端正常运行：浏览器访问 `http://公网IP:8080/api/job/list` 能返回 JSON 数据。

## 五、验证

1. 浏览器访问 `http://ECS公网IP`
2. F12 打开开发者工具 → Network 面板
3. 刷新页面，确认：
   - 静态资源从 ECS 加载（状态码 200）
   - API 请求发往本地公网 IP:8080（状态码 200）
   - WebSocket 连接正常建立
4. 跑 Lighthouse 审计查看性能分数

## 六、常见问题

### API 请求 404 / 连接超时
- 检查本地 8080 端口是否对外可访问
- 在另一台设备上访问 `http://本地公网IP:8080/api/job/list` 测试
- 如果是运营商封锁了 8080，换一个端口（如 9090），同步修改 `.env.production` 和后端配置

### WebSocket 连接失败
- WebSocket 用的是同一个 8080 端口，确保防火墙和端口转发都已放行
- 如果 ECS 用的是 HTTPS，WebSocket 需要用 `wss://`，需要额外配置 SSL

### 页面刷新后 404
- nginx 的 `try_files` 没配对，确认 location 块里有 `try_files $uri $uri/ /index.html`

### 本地电脑 IP 变了
- 家庭宽带的公网 IP 可能会变，每次变了需要更新 `.env.production` 重新打包
- 解决方案：用 DDNS 动态域名，或部署到云服务器
