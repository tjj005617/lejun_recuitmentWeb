# 消息推送 — 设计思路与实现

## 一、设计思路

### 1.1 核心问题

招聘平台需要在用户不停留在消息页面时，也能实时收到新消息通知。具体场景：

- 用户在浏览职位，HR 发来消息 → 需要弹通知
- 用户在首页，候选人投递了简历 → HR 需要收到通知
- 用户在任意页面，有人发起视频面试邀请 → 需要弹窗提醒

### 1.2 方案选型

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| 轮询（Polling） | 实现简单 | 延迟高、浪费带宽 | 仅作兜底 |
| WebSocket | 全双工 | 需要额外连接管理 | 已被聊天模块占用 |
| **SSE（Server-Sent Events）** | HTTP 长连接、自动重连、轻量 | 单向（服务端→客户端） | **主方案** |

选择 SSE 的原因：
- 消息推送是**单向**的（服务端通知客户端），SSE 完美匹配
- 复用 HTTP 协议，不需要额外的 WebSocket 端点
- 浏览器原生支持自动重连
- 前端已有聊天 WebSocket，避免再维护一个 WS 连接

### 1.3 双通道保障

```
主通道: SSE（实时推送，延迟 < 100ms）
  ↕ 自动切换
兜底通道: 轮询（SSE 断开时启用，每 5 秒检查一次）
```

SSE 连接成功 → 停止轮询
SSE 连接失败/断开 → 启动轮询兜底

### 1.4 未读数存储：Redis

使用 Redis 原子操作维护未读数，避免每次查询数据库：

```
Key:   message:unread:{userId}
Value: 未读消息数量（整数）
操作:  INCR / DECRBY / GET
```

### 1.5 存储分工

| 数据 | 存储位置 | 原因 |
|------|----------|------|
| 消息通知（未读数、通知弹窗） | MySQL `message` 表 | 需要与用户表关联查询昵称 |
| 未读数计数 | Redis `message:unread:{userId}` | 原子操作，避免每次查 DB |

### 2.3 SSE 连接管理（SseEmitterManager）

基于 `ConcurrentHashMap<Long, SseEmitter>` 的连接管理器，每个用户只保留一个 SSE 连接：

```java
@Slf4j
@Component
public class SseEmitterManager {

    // userId → SseEmitter，每个用户只保留一个连接
    private final ConcurrentHashMap<Long, SseEmitter> emitters = new ConcurrentHashMap<>();

    // 注册新连接（自动替换旧连接）
    public void addEmitter(Long userId, SseEmitter emitter) {
        SseEmitter old = emitters.put(userId, emitter);
        if (old != null) {
            old.complete(); // 关闭旧连接
            log.info("用户{}的旧SSE连接已关闭", userId);
        }
        log.info("用户{}的SSE连接已注册，当前在线{}", userId, emitters.size());
    }

    // 安全移除：只在 emitter 是当前引用时才移除
    // 防止用户快速重连时，旧连接的回调误删新连接
    public void removeEmitterIfCurrent(Long userId, SseEmitter emitter) {
        emitters.remove(userId, emitter); // 两参数 remove，精确匹配对象引用
    }

    // 推送事件给指定用户
    public void sendToUser(Long userId, String eventName, Object data) {
        SseEmitter emitter = emitters.get(userId);
        if (emitter == null) {
            log.warn("用户{}不在线，SSE推送跳过", userId);
            return;
        }
        try {
            emitter.send(SseEmitter.event().name(eventName).data(data));
        } catch (IOException e) {
            log.warn("SSE推送失败，用户{}: {}", userId, e.getMessage());
            removeEmitterIfCurrent(userId, emitter); // 连接已死，清理
        }
    }

    public int getOnlineCount() {
        return emitters.size();
    }
}
```

关键设计点：
- **单连接约束**：同一用户只维持一个 SSE 连接，新连接自动替换旧连接
- **CAS 式移除**：`removeEmitterIfCurrent` 使用 `ConcurrentHashMap.remove(key, value)` 两参数版本，只在 value 是同一个对象引用时才移除，防止旧回调误删新连接
- **Fire-and-forget**：推送失败被 catch 吞掉，不影响业务事务

### 2.4 SSE 端点（SseController）

```java
@Slf4j
@RestController
public class SseController {

    private final JwtUtil jwtUtil;
    private final SseEmitterManager sseEmitterManager;

    @GetMapping(value = "/message/subscribe", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public SseEmitter subscribe(@RequestParam("token") String token) {
        // 1. JWT 验证
        Long userId = jwtUtil.getUserId(token);
        if (userId == null) {
            SseEmitter errorEmitter = new SseEmitter(0L);
            errorEmitter.completeWithError(new RuntimeException("token无效"));
            return errorEmitter;
        }

        // 2. 防止代理缓冲（nginx 等）
        ServletRequestAttributes attrs = (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
        HttpServletResponse response = attrs.getResponse();
        response.setHeader("X-Accel-Buffering", "no");
        response.setHeader("Cache-Control", "no-cache, no-store, must-revalidate");
        response.setHeader("Connection", "keep-alive");

        // 3. 创建 SseEmitter（timeout=0 表示永不超时）
        SseEmitter emitter = new SseEmitter(0L);

        // 4. 注册连接（自动替换旧连接）
        sseEmitterManager.addEmitter(userId, emitter);

        // 5. 注册清理回调
        emitter.onCompletion(() -> sseEmitterManager.removeEmitterIfCurrent(userId, emitter));
        emitter.onTimeout(() -> sseEmitterManager.removeEmitterIfCurrent(userId, emitter));
        emitter.onError(e -> sseEmitterManager.removeEmitterIfCurrent(userId, emitter));

        // 6. 发送初始心跳
        try {
            emitter.send(SseEmitter.event().name("connected").data("ok"));
        } catch (IOException e) {
            log.warn("SSE初始心跳发送失败: {}", e.getMessage());
        }

        return emitter;
    }
}
```

### 2.5 Redis 未读数缓存（NotificationCacheServiceImpl）

```java
@Slf4j
@Service
public class NotificationCacheServiceImpl implements NotificationCacheService {

    private static final String UNREAD_KEY_PREFIX = "message:unread:";
    private final StringRedisTemplate stringRedisTemplate;
    private final MessageMapper messageMapper;

    private String getKey(Long userId) {
        return UNREAD_KEY_PREFIX + userId;
    }

    // 收到消息时 +1（原子操作）
    @Override
    public void incrementUnread(Long userId) {
        stringRedisTemplate.opsForValue().increment(getKey(userId));
    }

    // 标记已读时 -N（原子操作）
    @Override
    public void decrementUnread(Long userId, int count) {
        if (count <= 0) return;
        Long result = stringRedisTemplate.opsForValue().decrement(getKey(userId), count);
        // 防负数：DB 和 Redis 可能不一致
        if (result != null && result < 0) {
            stringRedisTemplate.opsForValue().set(getKey(userId), "0");
        }
    }

    // 读取未读数（Redis 优先，DB 兜底）
    @Override
    public int getUnreadCount(Long userId) {
        // 1. 尝试 Redis
        String val = stringRedisTemplate.opsForValue().get(getKey(userId));
        if (val != null) {
            return Integer.parseInt(val); // 缓存命中
        }

        // 2. Redis 缓存未命中 → 查 MySQL
        int count = messageMapper.selectCount(new LambdaQueryWrapper<Message>()
                .eq(Message::getReceiverId, userId)
                .eq(Message::getIsRead, 0));

        // 3. 回填 Redis（无 TTL，持久存在直到被修改）
        stringRedisTemplate.opsForValue().set(getKey(userId), String.valueOf(count));
        return count;
    }

    @Override
    public void clearUnread(Long userId) {
        stringRedisTemplate.delete(getKey(userId));
    }
}
```

### 2.6 发送消息 + 推送（MessageServiceImpl）

```java
@Slf4j
@Service
public class MessageServiceImpl implements MessageService {

    private final MessageMapper messageMapper;
    private final UserMapper userMapper;
    private final JobMapper jobMapper;
    private final SseEmitterManager sseEmitterManager;
    private final NotificationCacheService notificationCacheService;

    @Override
    @Transactional(rollbackFor = Exception.class)
    public Message sendMessage(Long senderId, Message message) {
        // 1. 设置字段并入库（isRead=0，始终未读）
        message.setSenderId(senderId);
        message.setIsRead(0);
        messageMapper.insert(message);

        // 2. Redis 未读数 +1
        notificationCacheService.incrementUnread(message.getReceiverId());

        // 3. 查询发送者昵称
        User sender = userMapper.selectById(senderId);
        String senderName = null;
        if (sender != null) {
            senderName = sender.getNickname() != null ? sender.getNickname() : sender.getUsername();
        }

        // 4. 构建通知 payload
        Map<String, Object> notification = new HashMap<>();
        notification.put("senderId", senderId);
        notification.put("content", message.getContent());
        notification.put("jobId", message.getJobId());
        notification.put("type", message.getType());
        if (senderName != null) {
            notification.put("senderName", senderName);
        }

        // 5. SSE 推送（失败不影响事务，异常被 SseEmitterManager 吞掉）
        sseEmitterManager.sendToUser(message.getReceiverId(), "message", notification);

        log.info("消息发送成功: sender={}, receiver={}, type={}", senderId, message.getReceiverId(), message.getType());
        return message;
    }
}
```

### 2.7 标记已读（Redis -N）

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void markAsRead(Long receiverId, Long senderId) {
    // 1. 统计未读数量
    int unreadCount = messageMapper.selectCount(new LambdaQueryWrapper<Message>()
            .eq(Message::getReceiverId, receiverId)
            .eq(Message::getSenderId, senderId)
            .eq(Message::getIsRead, 0));

    // 2. 批量标记已读
    messageMapper.update(null, new LambdaUpdateWrapper<Message>()
            .eq(Message::getReceiverId, receiverId)
            .eq(Message::getSenderId, senderId)
            .eq(Message::getIsRead, 0)
            .set(Message::getIsRead, 1));

    // 3. Redis 未读数 -N
    if (unreadCount > 0) {
        notificationCacheService.decrementUnread(receiverId, unreadCount);
    }
    log.info("标记已读: receiver={}, sender={}, count={}", receiverId, senderId, unreadCount);
}
```

### 3.2 消息 Store（模块级共享状态）

```javascript
// frontend/src/stores/message.js
import { ref } from 'vue'
import { getUnreadCount, getLatestUnreadMessage, getConversations,
         getConversation, markAsRead } from '@/api/message'
import { ElNotification } from 'element-plus'

/** 模块级共享状态（所有组件共享同一份引用） */
const unreadCount = ref(0)
const conversations = ref([])
let eventSource = null
let reconnectTimer = null
let pollingTimer = null
let previousUnreadCount = 0
const selectedUserId = ref(null)
const wsConnected = ref(false)
const videoInviteHandledByWs = ref(false)
let videoInvitePopupShown = false
let onVideoInvite = null
let pendingVideoInvites = []

/** 连接 SSE */
const connectSSE = () => {
  if (reconnectTimer) { clearTimeout(reconnectTimer); reconnectTimer = null }

  const token = localStorage.getItem('token')
  if (!token) return

  // CONNECTING 超时检测：超过5秒还在连接中，强制重连
  if (eventSource) {
    if (eventSource.readyState === EventSource.OPEN) return // 已连接
    if (eventSource.readyState === EventSource.CONNECTING) {
      if (eventSource._connectTime && Date.now() - eventSource._connectTime > 5000) {
        try { eventSource.close() } catch {}
        eventSource = null
      } else {
        return // 还在连接中，等一下
      }
    }
  }

  fetchUnreadCount()

  const apiBase = import.meta.env.VITE_API_BASE_URL || '/api'
  const es = new EventSource(`${apiBase}/message/subscribe?token=${token}`)
  es._connectTime = Date.now()

  // 收到新消息 → 弹通知（不在消息页时）+ 刷新未读数
  es.addEventListener('message', (e) => {
    try {
      const data = JSON.parse(e.data)
      if (!window.location.pathname.includes('/messages')) {
        showNotification(data)
      }
      fetchUnreadCount() // 实时刷新未读数 badge
    } catch {}
  })

  // 收到视频邀约
  es.addEventListener('video_invite', (e) => {
    try {
      const data = JSON.parse(e.data)
      if (videoInvitePopupShown || videoInviteHandledByWs.value) return // 去重
      if (onVideoInvite) {
        videoInvitePopupShown = true
        onVideoInvite(data)
      } else {
        pendingVideoInvites.push({ data, _ts: Date.now() }) // 暂存，等回调注册后处理
      }
    } catch {}
  })

  // 连接成功 → 停止轮询
  es.addEventListener('open', () => {
    fetchUnreadCount()
    stopPolling()
    checkPendingVideoInvite() // 检查是否有未处理的视频邀约
  })

  // 连接失败 → 启动轮询兜底
  es.onerror = () => {
    startPolling()
    try { es.close() } catch {}
    eventSource = null
    reconnectTimer = setTimeout(connectSSE, 3000) // 3秒后重连
  }

  eventSource = es
}

/** 断开 SSE */
const disconnectSSE = () => {
  if (reconnectTimer) { clearTimeout(reconnectTimer); reconnectTimer = null }
  stopPolling()
  if (eventSource) { try { eventSource.close() } catch {}; eventSource = null }
}

/** 拉取数据库真实未读数 */
const fetchUnreadCount = async () => {
  try {
    const res = await getUnreadCount()
    unreadCount.value = res.data ?? 0
    previousUnreadCount = unreadCount.value
  } catch {}
}

/** 轮询兜底：SSE 不可用时每 5 秒检查一次 */
const startPolling = () => {
  if (pollingTimer) return
  let tickCount = 0
  pollingTimer = setInterval(async () => {
    tickCount++
    try {
      const res = await getUnreadCount()
      const newCount = res.data ?? 0
      // 未读数增加 → 查最新未读消息
      if (newCount > previousUnreadCount) {
        const latestRes = await getLatestUnreadMessage()
        const latest = latestRes.data
        if (latest) {
          if (latest.type === 'video_invite' && onVideoInvite
              && !videoInviteHandledByWs.value && !videoInvitePopupShown) {
            videoInvitePopupShown = true
            onVideoInvite({ type: 'video_invite', senderId: latest.senderId,
              jobId: latest.jobId || null, conversationId: null })
          } else if (latest.type !== 'video_invite'
              && !window.location.pathname.includes('/messages')) {
            showNotification(latest)
          }
        }
      }
      // 每6轮（30秒）定期检查待处理的视频邀约
      if (tickCount % 6 === 0 && onVideoInvite
          && !videoInviteHandledByWs.value && !videoInvitePopupShown) {
        try {
          const latestRes = await getLatestUnreadMessage()
          const latest = latestRes.data
          if (latest && latest.type === 'video_invite') {
            videoInvitePopupShown = true
            onVideoInvite({ type: 'video_invite', senderId: latest.senderId,
              jobId: latest.jobId || null, conversationId: null })
          }
        } catch {}
      }
      unreadCount.value = newCount
      previousUnreadCount = newCount
    } catch {}
  }, 5000)
}

/** 停止轮询 */
const stopPolling = () => {
  if (pollingTimer) { clearInterval(pollingTimer); pollingTimer = null }
}

/** 通知去重：5秒内相同内容不重复弹 */
let lastNotifyKey = ''
let lastNotifyTime = 0
const showNotification = (data) => {
  const senderName = data.senderName || '对方'
  const content = data.content || '您有一条新消息'
  const key = senderName + '|' + content
  const now = Date.now()
  if (key === lastNotifyKey && now - lastNotifyTime < 5000) return
  lastNotifyKey = key
  lastNotifyTime = now
  ElNotification({
    title: `${senderName} 发来消息`,
    message: content,
    type: 'info',
    duration: 5000,
    customClass: 'notify-message'
  })
}
```

## 五、关键设计决策

| 决策 | 原因 |
|------|------|
| SSE 作为主通道 | 单向推送场景最合适，比 WS 轻量，比轮询实时 |
| Redis 存未读数 | 原子操作避免并发问题，避免频繁查 DB |
| 轮询兜底 | SSE 在某些网络/代理环境下不稳定，轮询保障可达性 |
| 推送失败不回滚事务 | 通知是辅助功能，不能因为推送失败导致消息发送失败 |
| 5 秒通知去重 | 防止网络抖动导致同一消息多次推送弹窗 |
| 聊天在线时不推通知 | 用户正在看聊天页面时，不需要弹通知，避免未读数虚高 |
| CAS 式移除连接 | 防止用户快速重连时旧回调误删新连接 |
| SSE 收到消息立即刷新未读数 | SSE 是实时通道，收到推送就该同步更新 badge，避免"弹了通知但数字不变" |
