# 俊乐招聘网站 - 需求分析与功能设计

## 一、系统概述

俊乐招聘网站是一个连接求职者和招聘方的在线平台，主要功能包括：
- 求职者：搜索职位、投递简历、收藏职位、接收消息
- HR/面试官：发布职位、管理申请、查看候选人、发送消息
- AI模拟面试：上传简历，AI智能出题，助用户面试准备

---

## 三、API接口详细设计

### 3.1 用户模块（已完成）

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 注册 | POST | `/api/user/register` | 支持roleType选择 |
| 登录 | POST | `/api/user/login` | 返回用户信息含roleType |
| 获取用户信息 | GET | `/api/user/{id}` | 含手机号、邮箱 |
| 获取用户简历 | GET | `/api/user/{id}/resumes` | 返回简历ID列表 |
| 添加简历 | POST | `/api/user/{id}/resumes` | 最多3份 |
| 删除简历 | DELETE | `/api/user/{id}/resumes/{resumeId}` | 移除关联 |

### 3.2 职位模块

#### 3.2.1 发布职位
- **URL**: `POST /api/job`
- **权限**: HR（roleType=2）

**请求参数:**
```json
{
  "title": "Java开发工程师",
  "categoryId": 6,
  "city": "北京",
  "address": "海淀区中关村",
  "jobType": "full_time",
  "experience": "3-5年",
  "education": "本科",
  "salaryMin": 15000,
  "salaryMax": 30000,
  "description": "负责公司核心业务系统开发...",
  "requirements": "1. 3年以上Java开发经验...",
  "benefits": "[\"五险一金\",\"带薪年假\"]"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | String | 是 | 职位名称 |
| categoryId | Long | 否 | 职位分类ID |
| city | String | 是 | 工作城市 |
| address | String | 否 | 工作地址 |
| jobType | String | 是 | full_time/part_time/intern |
| experience | String | 否 | 经验要求 |
| education | String | 否 | 学历要求 |
| salaryMin | Integer | 否 | 薪资下限（元/月） |
| salaryMax | Integer | 否 | 薪资上限（元/月） |
| description | String | 是 | 职位描述（富文本） |
| requirements | String | 否 | 任职要求（富文本） |
| benefits | String | 否 | 福利JSON数组 |

**成功响应:**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "title": "Java开发工程师",
    "companyId": 1,
    "createdAt": "2026-06-15T10:00:00"
  }
}
```

#### 3.4.5 更新申请状态
- **URL**: `PUT /api/application/{id}/status`
- **权限**: HR

**请求参数:**
```json
{
  "status": "screening",
  "hrRemark": "简历已查看，符合要求",
  "rejectReason": null
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | String | 是 | 新状态 |
| hrRemark | String | 否 | HR备注 |
| rejectReason | String | 否 | 拒绝原因（status=rejected时） |

**状态流转:**
```
pending（待处理）
  ↓
screening（筛选中）
  ↓
interview（面试中）
  ↓
offer（已录用）/ rejected（已拒绝）
```

## 四、状态码定义

### 4.1 申请状态
| 状态 | 说明 | 可流转到 |
|------|------|----------|
| pending | 待处理（默认） | screening, withdrawn |
| screening | 筛选中 | interview, rejected |
| interview | 面试中 | offer, rejected |
| offer | 已录用 | - |
| rejected | 已拒绝 | - |
| withdrawn | 已撤回 | - |

### 4.2 职位状态
| 状态 | 说明 |
|------|------|
| active | 招聘中（默认） |
| paused | 已暂停 |
| closed | 已关闭 |

### 4.3 消息类型
| 类型 | 说明 |
|------|------|
| chat | 聊天消息 |
| application | 申请相关通知 |
| system | 系统通知 |

## 五、页面与功能对照

| 页面 | 路径 | 功能 | 依赖接口 |
|------|------|------|----------|
| 首页 | `/` | 搜索、热门职位、热门公司 | /job/hot, /company/hot |
| 职位搜索 | `/jobs` | 关键词搜索、筛选 | /job/list |
| 职位详情 | `/job/:id` | 查看详情、投递、收藏 | /job/{id}, /application, /favorite |
| 公司详情 | `/company/:id` | 公司信息、在招职位 | /company/{id}, /company/{id}/jobs |
