# 聊天功能 — 设计思路与实现

## 一、设计思路

### 1.1 核心需求

HR 和求职者之间需要实时沟通，支持：
- 文字消息收发
- 简历附件发送
- 消息已读回执
- 聊天记录持久化与历史回放
- 消息防重复
- Emoji 表情和语音输入

### 1.2 存储方案：MySQL + MongoDB 混合

| 数据 | 存储位置 | 原因 |
|------|----------|------|
| 消息通知（未读数、通知弹窗） | MySQL `message` 表 | 需要与业务数据（用户、职位）关联查询 |
| 聊天消息（文字、简历、视频信令） | MongoDB `chat_message` 集合 | 写入量大、文档模型灵活、不需要复杂 JOIN |

**为什么不用单一数据库？**
- MySQL 适合结构化查询（会话列表、未读数统计），但高频写入聊天消息会有性能瓶颈
- MongoDB 适合高频追加写入和灵活的文档结构（简历附件、视频信令等不同类型消息）
- 两者各司其职：MySQL 管通知，MongoDB 管聊天记录

### 1.3 实时通信：复用聊天 WebSocket

所有聊天消息通过已有的 `ws/chat` WebSocket 传输，不引入新的连接：

```
连接地址: ws://{host}/ws/chat?token={jwtToken}&conversationId={userId1}_{userId2}
conversationId: 两个用户ID拼接，小ID在前（如 "3_5"）
```

### 1.4 消息去重：两层防护

| 层级 | 机制 | 防护场景 |
|------|------|----------|
| 内存层 | WebSocket `recentMessages` Map，5 秒 TTL | 客户端连击发送按钮 |
| 持久层 | MongoDB 5 字段精确匹配查询 | 网络重试、事务重复提交 |

### 1.5 前端防重：内容指纹 + 频率检测

前端还有一层防重机制：
- **内容指纹**：相同内容 3 秒内不重复发送
- **频率检测**：3 秒内发送达到 3 条 → 锁定 5 秒
- **乐观更新**：先本地显示临时消息，等后端广播回来替换为真实 ID

```java
// ChatServiceImpl.java — 保存聊天消息（含 MongoDB 幂等检查）
public ChatMessage saveMessage(ChatMessage message) {
    if (message.getCreatedAt() == null) {
        message.setCreatedAt(LocalDateTime.now());
    }
    if (message.getRead() == null) {
        message.setRead(false);
    }
    // 自动生成会话ID
    if (message.getConversationId() == null) {
        message.setConversationId(
            ChatMessage.generateConversationId(message.getSenderId(), message.getReceiverId())
        );
    }

    // 幂等性检查：5字段精确匹配 → 防止重复写入
    Query dedupQuery = new Query(Criteria
            .where("conversationId").is(message.getConversationId())
            .and("senderId").is(message.getSenderId())
            .and("content").is(message.getContent())
            .and("type").is(message.getType())
            .and("createdAt").is(message.getCreatedAt()));
    ChatMessage existing = mongoTemplate.findOne(dedupQuery, ChatMessage.class);
    if (existing != null) {
        log.warn("检测到重复消息，跳过保存: existingId={}", existing.getId());
        return existing;
    }

    return chatMessageRepository.save(message);
}

// 标记已读（MongoDB 批量更新）
public long markAsRead(String conversationId, Long receiverId) {
    Query query = new Query(
        Criteria.where("conversationId").is(conversationId)
            .and("receiverId").is(receiverId)
            .and("read").is(false)
    );
    Update update = new Update().set("read", true);
    var result = mongoTemplate.updateMulti(query, update, ChatMessage.class);
    return result.getModifiedCount();
}
```

### 2.4 后端 ChatWebSocket — 核心消息处理

```java
// ChatWebSocket.java — WebSocket 连接建立
public void afterConnectionEstablished(WebSocketSession session) throws Exception {
    // 1. 从 URL 参数解析 token 和 conversationId
    String query = session.getUri().getQuery();
    String token = null, conversationId = null;
    for (String param : query.split("&")) {
        String[] kv = param.split("=", 2);
        if ("token".equals(kv[0])) token = kv[1];
        if ("conversationId".equals(kv[0])) conversationId = kv[1];
    }

    // 2. JWT 验证
    Long userId = jwtUtil.getUserId(token);
    if (userId == null) { session.close(CloseStatus.NOT_ACCEPTABLE); return; }

    // 3. 设置消息大小限制（支持视频中继二进制帧）
    session.setTextMessageSizeLimit(1024 * 1024);     // 1MB 文本
    session.setBinaryMessageSizeLimit(2 * 1024 * 1024); // 2MB 二进制

    // 4. 注册会话（ConcurrentHashMap 保证线程安全）
    SessionInfo info = new SessionInfo(userId, conversationId);
    sessionInfoMap.put(session.getId(), info);
    conversationSessions.computeIfAbsent(conversationId, k -> ConcurrentHashMap.newKeySet()).add(session);

    // 5. 推送完整聊天历史
    List<ChatMessage> history = chatService.getHistory(conversationId);
    sendMessage(session, Map.of("type", "history", "messages", history));
}
```

```java
// ChatWebSocket.java — 消息路由（switch 分发）
protected void handleTextMessage(WebSocketSession session, TextMessage message) {
    SessionInfo info = sessionInfoMap.get(session.getId());
    Map<String, Object> payload = objectMapper.readValue(message.getPayload(), Map.class);
    String type = (String) payload.get("type");

    switch (type) {
        case "text" -> handleText(session, info, payload);           // 文字消息
        case "resume" -> handleResume(session, info, payload);       // 简历附件
        case "read" -> handleRead(session, info);                    // 已读标记
        case "ack" -> handleAck(session, info, payload);             // 消息确认
        case "video_invite", "video_accept", "video_reject", "video_hangup" ->
                handleVideoSignal(session, info, payload);           // 视频信令（广播）
        case "video_offer", "video_answer", "video_ice" ->
                handleVideoTargeted(session, info, payload);         // WebRTC 信令（定向）
    }
}
```

```java
// ChatWebSocket.java — 文字消息处理（核心流程）
private void handleText(WebSocketSession session, SessionInfo info, Map<String, Object> payload) {
    String content = (String) payload.get("content");
    if (content == null || content.trim().isEmpty()) return; // 拒绝空消息

    Long jobId = payload.get("jobId") != null ? Long.valueOf(payload.get("jobId").toString()) : null;

    // 1. 内存层去重（5秒窗口）
    String dedupKey = info.userId() + "|" + content;
    long now = System.currentTimeMillis();
    Long lastTime = recentMessages.put(dedupKey, now);
    if (lastTime != null && now - lastTime < 5000) {
        recentMessages.put(dedupKey, lastTime); // 恢复时间戳
        return;
    }

    // 2. 构建 ChatMessage
    Long receiverId = parseReceiverId(info.conversationId(), info.userId());
    ChatMessage chatMessage = new ChatMessage();
    chatMessage.setConversationId(info.conversationId());
    chatMessage.setSenderId(info.userId());
    chatMessage.setReceiverId(receiverId);
    chatMessage.setJobId(jobId);
    chatMessage.setType("text");
    chatMessage.setContent(content);
    chatMessage.setStatus("sent");

    // 3. MongoDB 持久化（含二次去重）
    ChatMessage saved = chatService.saveMessage(chatMessage);

    // 4. 广播给会话中的所有人
    broadcastToConversation(info.conversationId(), Map.of("type", "message", "data", saved));

    // 5. 条件通知：接收方不在聊天页时才推送
    if (!isUserInConversation(receiverId, info.conversationId())) {
        sendChatNotification(saved);
    }
}
```

```java
// ChatWebSocket.java — 广播：发给会话中的所有人
public void broadcastToConversation(String conversationId, Object message) {
    Set<WebSocketSession> sessions = conversationSessions.get(conversationId);
    if (sessions == null) return;
    String json = objectMapper.writeValueAsString(message);
    for (WebSocketSession session : sessions) {
        if (session.isOpen()) {
            session.sendMessage(new TextMessage(json));
        }
    }
}

// 定向：只发给指定用户（WebRTC offer/answer/ice）
public void sendToUser(Long userId, String conversationId, Object message) {
    Set<WebSocketSession> sessions = conversationSessions.get(conversationId);
    if (sessions == null) return;
    String json = objectMapper.writeValueAsString(message);
    for (WebSocketSession session : sessions) {
        SessionInfo info = sessionInfoMap.get(session.getId());
        if (info != null && info.userId().equals(userId) && session.isOpen()) {
            session.sendMessage(new TextMessage(json));
        }
    }
}

// 二进制帧定向发送（视频中继专用）
public void sendBinaryToUser(Long userId, String conversationId, java.nio.ByteBuffer data) {
    Set<WebSocketSession> sessions = conversationSessions.get(conversationId);
    if (sessions == null) return;
    for (WebSocketSession session : sessions) {
        SessionInfo info = sessionInfoMap.get(session.getId());
        if (info != null && info.userId().equals(userId) && session.isOpen()) {
            data.rewind(); // 多 session 时确保 position 正确
            session.sendMessage(new BinaryMessage(data));
        }
    }
}
```

```java
// 判断用户是否在聊天页（有 WebSocket 连接）
public boolean isUserInConversation(Long userId, String conversationId) {
    Set<WebSocketSession> sessions = conversationSessions.get(conversationId);
    if (sessions == null) return false;
    for (WebSocketSession session : sessions) {
        SessionInfo info = sessionInfoMap.get(session.getId());
        if (info != null && info.userId().equals(userId) && session.isOpen()) {
            return true;
        }
    }
    return false;
}
```

```java
private void handleRead(WebSocketSession session, SessionInfo info) {
    // 1. MongoDB 批量标记已读
    long count = chatService.markAsRead(info.conversationId(), info.userId());

    // 2. Redis 未读数 -N
```

```java
public void afterConnectionClosed(WebSocketSession session, CloseStatus status) {
    SessionInfo info = sessionInfoMap.remove(session.getId());
    if (info != null) {
        Set<WebSocketSession> sessions = conversationSessions.get(info.conversationId());
        if (sessions != null) {
            sessions.remove(session);
            if (sessions.isEmpty()) {
                conversationSessions.remove(info.conversationId());
            }
        }
    }
}
```

### 2.10 前端：WebSocket 连接管理

```javascript
// MessageCenter.vue — 建立 WebSocket 连接
const connectWebSocket = (conversationId) => {
  disconnectWebSocket()
  currentConversationId = conversationId

  const token = localStorage.getItem('token')
  const protocol = location.protocol === 'https:' ? 'wss:' : 'ws:'
  const wsHost = import.meta.env.VITE_WS_HOST || location.host
  const wsUrl = `${protocol}//${wsHost}/ws/chat?token=${token}&conversationId=${conversationId}`
  ws = new WebSocket(wsUrl)
  ws.binaryType = 'arraybuffer' // 接收二进制帧（视频中继）

  ws.onmessage = (event) => {
    // 二进制帧：视频中继数据（跳过 JSON 解析）
    if (event.data instanceof ArrayBuffer) {
      handleBinaryVideoFrame(event.data)
      return
    }

    const data = JSON.parse(event.data)
    switch (data.type) {
      case 'history':
        // 按唯一ID去重后加载历史
        messages.value = (data.messages || []).filter((m, i, arr) => {
          const mid = m.id || m._id
          return arr.findIndex(x => (x.id || x._id) === mid) === i
        })
        loadResumesForMessages(messages.value)
        nextTick(scrollToBottom)
        break
      case 'message':
        // 移除乐观更新的临时消息，ID去重后追加真实消息
        const msg = data.data
        messages.value = messages.value.filter(m => {
          const isTemp = String(m.id || m._id).startsWith('temp_')
          return !(isTemp && m.content === msg.content && m.senderId === msg.senderId)
        })
        if (!messages.value.some(m => (m.id || m._id) === (msg.id || msg._id))) {
          messages.value.push(msg)
        }
        nextTick(scrollToBottom)
        // 接收方发送 ACK 确认
        if (msg.senderId !== currentUserId.value && (msg.id || msg._id)) {
          wsSend({ type: 'ack', messageId: msg.id || msg._id })
        }
        break
      case 'ack':
        // 发送方收到确认，更新消息状态
        const ackMsg = messages.value.find(m => (m.id || m._id) === data.messageId)
        if (ackMsg) ackMsg.status = 'sent'
        break
      case 'read':
        // 对方已读（可选：显示已读标记）
        break
    }
  }

  ws.onopen = () => { messageStore.wsConnected.value = true }
  ws.onclose = () => { ws = null; messageStore.wsConnected.value = false }
}
```

### 2.11 前端：发送消息（防重 + 乐观更新）

```javascript
// MessageCenter.vue — 发送文字消息
const isSending = ref(false)
let lastSentContent = ''
let lastSentTime = 0
let sendCount = 0
let sendWindowStart = 0

const sendTextMessage = () => {
  if (isSending.value) return

  const snapshot = newMessage.value
  const content = snapshot.trim()
  if (!content || !selectedUserId.value) return

  const now = Date.now()

  // 核心防重：相同内容 3 秒内绝不重复发送
  if (content === lastSentContent && now - lastSentTime < 3000) return

  // 频率检测：3秒内发送达到3条 → 锁定5秒
  if (now - sendWindowStart > 3000) { sendCount = 0; sendWindowStart = now }
  sendCount++
  if (sendCount >= 3) {
    isSending.value = true
    ElMessage.warning('发送过于频繁，请稍后再试')
    setTimeout(() => { isSending.value = false }, 5000)
    return
  }

  // 先锁住再发送
  isSending.value = true
  lastSentContent = content
  lastSentTime = now
  newMessage.value = ''

  // 乐观更新：先本地显示临时消息
  const tempId = 'temp_' + Date.now()
  messages.value.push({
    _id: tempId, id: tempId,
    senderId: currentUserId.value,
    content, type: 'text',
    createdAt: new Date().toISOString()
  })
  nextTick(scrollToBottom)

  // 通过 WebSocket 发送
  wsSend({ type: 'text', content })

  // 200ms后解锁
  setTimeout(() => { isSending.value = false }, 200)
}
```

## 六、关键设计决策

| 决策 | 原因 |
|------|------|
| MySQL + MongoDB 混合存储 | MySQL 管通知/关联查询，MongoDB 管高频聊天写入 |
| conversationId 小ID在前 | 保证同一对用户映射唯一会话，避免双向查找 |
| 两层去重（内存+MongoDB） | 内存层快速拦截，持久层兜底防并发 |
| 条件通知 | 用户在线时不推通知，避免未读数虚高 |
| 广播 vs 定向 | 信令类广播（invite/accept/reject），WebRTC类定向（offer/answer/ice） |
| 定时清理去重 Map | 每 60 秒清理 10 秒前的条目，防止内存泄漏 |
| 乐观更新 | 先本地显示临时消息，用户体验流畅，等后端广播回来替换为真实ID |
| 前端三重防重 | 内容指纹 + 频率检测 + 锁定机制，杜绝重复发送 |
| binaryType=arraybuffer | WebSocket 同时支持文本（聊天）和二进制（视频中继）帧 |
| 条件通知（后端） | 后端也检查接收方是否在线，与前端配合双重保障 |
